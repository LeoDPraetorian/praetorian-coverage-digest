## Key Quotes

### On Server vs Client State

> "For a vast majority of applications, the truly globally accessible client state that is left over after migrating all of your async code to TanStack Query is usually very tiny." - TanStack Query Docs

### On Zustand Decision

> "Start with Context. Add TanStack Query for server data. Only reach for Zustand when you have a specific problem that Context doesn't solve." - State Management in 2025, DEV Community

### On Context Performance

> "React Compiler typically delivers 30-60% reduction in unnecessary re-renders... mirrors the benefits of libraries like `use-context-selector` without extra code." - Meta React Compiler Announcement, InfoQ

### On Optimistic Updates

> "Optimistic updates are a bit over-used. Not every mutation needs to be done optimistically. You should really be sure that it rarely fails, because the UX for a rollback is not great." - TkDodo, Mastering Mutations in React Query

### On The 90% Rule

> "For 90% of SaaS platforms, MVPs, and enterprise dashboards, Zustand is the default choice... But start with Context." - Meerako, React State Management 2025

### On Simplicity

> "The best state management solution is the simplest one that works." - React State Management in 2025, DeveloperWay
